## 2. SecuritySettingsViewController（账号安全主页）增强

### 2.1 现有状态
- 4 个列表行：手机号、登录密码、微信授权、注销账户
- 顶部状态卡片：根据手机号是否绑定显示"安全状态良好"或"建议绑定"
- 从 `UserManager.shared.currentUser` 读取数据
- 手机号脱敏显示（`138****5678`）

### 2.2 需要调整
- **登录密码**：当前只显示提示文字，需改为从 `UserManager.shared.currentUser` 判断密码状态（字段 `pwd` 是否存在）
- **微信授权**：当前用 `fd_wechat_nickname` UserDefaults 模拟，需改为判断 `SUsers.openIdWechat` 字段
- **状态卡片文案**：根据手机号 + 密码 + 微信三个维度综合判断

### 2.3 字段映射

| UI 元素 | 数据来源 | 展示逻辑 |
|---|---|---|
| 手机号 value | `currentUser.mobile` | 脱敏 `138****5678`，点击 → `/me/change-phone` |
| 登录密码 value | `currentUser.pwd` | 有值 → "已设置"，nil/空 → "设置密码" |
| 微信授权 value | `currentUser.openIdWechat` | 有值 → 微信昵称，nil → "未绑定" |
| 状态卡片标题 | 综合判断 | 三项都有 → "账号安全状态良好"，缺项 → "建议完善账号安全" |

## 3. ChangePhoneViewController（手机号修改）— 新建

路由: `/me/change-phone`

### 3.1 页面布局
```
┌─ NavBar: 更换手机号 ← ──────────────────┐
│                                          │
│  更换后，可使用新手机号登录富德健康。        │
│                                          │
│  当前手机号                               │
│  138****5678                             │
│                                          │
│  ┌─ 新手机号 ──────────────────────────┐  │
│  │ [________________________] 11位     │  │
│  └────────────────────────────────────┘  │
│                                          │
│  ┌─ 验证码 ────────────────────────────┐  │
│  │ [________] [发送验证码]              │  │
│  └────────────────────────────────────┘  │
│  演示验证码：111111                       │
│                                          │
│  ☐ 我已阅读并同意《用户协议》《隐私政策》    │
│     与《健康管理服务知情同意书》            │
│                                          │
│  [────────── 确认更换 ──────────]         │
│                                          │
└──────────────────────────────────────────┘
```

### 3.2 校验规则

| 字段 | 规则 | 异常提示 |
|---|---|---|
| 新手机号 | 11 位，`1[3-9]\d{9}` | "请输入正确的手机号" |
| 验证码 | 6 位数字 | "请输入6位验证码" / "验证码错误，请重新输入" |
| 协议勾选 | 必须勾选 | "请先阅读并同意用户协议、隐私政策与健康管理服务知情同意书"，协议区域高亮抖动 |

### 3.3 交互流程

1. 用户输入新手机号 → 点击"发送验证码"
2. 手机号格式校验 → 不通过: Toast 提示
3. 调用 `LoginService.sendSmsCode(phone)` → 开始 60s 倒计时
4. 用户输入验证码 → 点击"确认更换"
5. 校验验证码 + 协议 → 不通过: 对应提示
6. 调用 `UserService.changeMobile(oldMobile:newMobile:checkCode:)`
7. 成功 → Toast "手机号已更换" → 更新 `UserDefaults "current_user_mobile"` → `UserManager.shared.refreshUserInfo()` → `router.back()`

### 3.4 协议弹窗

点击协议链接 → ActionSheet 弹出完整协议文本（占位，正式上线替换）

## 4. PasswordSetupViewController（登录密码设置）— 新建

路由: 内嵌在 SecuritySettingsViewController 中（step 模式）或独立页面

### 4.1 三步流程

#### Step 1: 手机验证
```
┌─ 手机验证 ──────────────────────────────┐
│  通过短信验证码确认身份后，可设置新的      │
│  登录密码。                              │
│                                         │
│  手机号                                  │
│  ┌─ 📱 [________________] 11位 ───────┐ │
│  └────────────────────────────────────┘ │
│  演示手机号：15600000003                 │
│                                         │
│  [────────── 获取验证码 ──────────]      │
└─────────────────────────────────────────┘
```

#### Step 2: 填写验证码
```
┌─ 填写验证码 ────────────────────────────┐
│  验证码已发送至 156****0003              │
│                                         │
│  验证码                                  │
│  ┌─ 🛡 [________] [重新获取(60s)] ───┐  │
│  └────────────────────────────────────┘ │
│  演示验证码：111111                      │
│                                         │
│  [────────── 下一步 ──────────]          │
└─────────────────────────────────────────┘
```

#### Step 3: 设置新密码
```
┌─ 设置新密码 ────────────────────────────┐
│  建议 6-20 位，可用数字和字母组合。       │
│  请避免使用生日、手机号后 6 位等          │
│  容易被猜到的密码。                      │
│                                         │
│  新密码                                  │
│  ┌─ 🔒 [_______________] 👁 ─────────┐  │
│  └────────────────────────────────────┘ │
│                                         │
│  确认新密码                              │
│  ┌─ 🔒 [_______________] ─────────────┐  │
│  └────────────────────────────────────┘ │
│                                         │
│  [────────── 完成设置 ──────────]        │
└─────────────────────────────────────────┘
```

### 4.2 校验规则

| 步骤 | 字段 | 规则 | 异常提示 |
|---|---|---|---|
| Step 1 | 手机号 | 11 位 | "请输入正确的手机号" |
| Step 1 | 验证码发送 | 调用 `LoginService.sendSmsCode(phone)` | — |
| Step 2 | 验证码 | 6 位数字，校验 | "请输入6位验证码" / "验证码错误，请重新输入" |
| Step 3 | 新密码 | 6-20 位 | "新密码至少6位" / "新密码不能超过20位" |
| Step 3 | 确认密码 | 必须与新密码一致 | "两次输入的密码不一致，请重新输入" |

### 4.3 完成回调

调用 `UserService.changePassword(mobile:oldPwd:newPwd:checkCode:)`
- 首次设置（无旧密码）：`oldPwd = nil`
- 修改密码（已有密码）：`oldPwd = "旧密码"`

成功 → Toast "密码设置成功" → 返回账号安全页 → `UserManager.shared.refreshUserInfo()`

## 5. 微信授权管理 — 增强

### 5.1 点击逻辑

```
点击"微信授权"
  ├─ 未绑定（openIdWechat == nil）
  │    └─ 唤起微信 SDK 授权 → 成功: 更新 UserManager → Toast "微信已绑定"
  │                         → 取消: Toast "已取消微信授权"
  │                         → 失败: Toast "微信授权失败，请稍后重试"
  │
  └─ 已绑定（openIdWechat != nil）
       └─ 弹出确认弹窗:
           标题: "是否解绑微信账号？"
           说明: "解绑后，将无法使用微信快速登录富德健康。"
           [取消] [立即解绑]
              → 取消: 关闭弹窗
              → 立即解绑: 调用 API → Toast "微信已解绑" → refreshUserInfo()
```

### 5.2 V1.0 实现策略

微信 SDK 接入需要 AppID 和审核，V1.0 采用过渡方案：
- 绑定：模拟成功（像原型一样存 `fd_wechat_nickname`）
- 解绑：展示真实二次确认弹窗，清除模拟数据
- 预留对接真实微信 SDK 的接口

### 6.2 注销前订单检查

```
点击"申请注销"
  ├─ 查询订单: 是否存在 status ∈ {pending_use, in_progress, pending_review} 的订单
  │
  ├─ 存在未完成订单
  │    └─ 弹窗:
  │        标题: "暂无法注销账户"
  │        说明: "您当前还有未完成的订单或服务，请处理完成后再申请注销。"
  │        [我知道了] [查看订单]
  │          → 我知道了: 关闭弹窗，留在当前页
  │          → 查看订单: push /orders
  │
  └─ 无未完成订单
       └─ 弹窗:
           标题: "注销确认"
           说明: "注销后您将失去本账户的所有信息，请谨慎操作"
           [取消] [确定注销] (红色)
             → 取消: 关闭弹窗
             → 确定注销:
                 1. 调用注销 API（预留）
                 2. 清除登录态: LoginService.clearSession()
                 3. 清除用户缓存: UserManager.shared.clear()
                 4. 展示结果页 → 2s 后跳转 /login
```

### 6.3 订单检查实现

V1.0 使用已有订单数据：
- 从订单服务或本地缓存读取订单列表
- 检查是否有 status 为 `pending_use` / `in_progress` / `pending_review` 的订单
- 如果暂时没有订单服务，可先默认允许注销（直接到二次确认）

### 6.4 注销结果页

```
┌───────────────────────────────────────────┐
│                                           │
│              ✅ (绿色大图标)                │
│                                           │
│              注销成功                      │
│     您的账户已成功注销，即将跳转至          │
│     注册页面。                             │
│                                           │
└───────────────────────────────────────────┘
        2 秒后 → router.replace('/login')
```
