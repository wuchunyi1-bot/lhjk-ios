# Design: 订单列表待支付 → 确认页差异化展示

## 入口矩阵

| 来源 | 路由 `entry` | 导航标题 | 顶部状态区 | 底部订单信息 |
|------|-------------|---------|-----------|-------------|
| 购物车去结算 | `cart` | 确认订单 | 无 | 无 |
| 套餐立即下单 | 缺省 | 确认订单 | 无 | 无 |
| 我的订单待支付 | `order_pay` | **订单详情** | **待支付** | **有** |

## 页面结构（order_pay）

纵向顺序（对齐订单详情待支付语义 + 确认页可编辑区）：

1. **状态头** `OrderDetailStatusView`：主文案「待支付」，副文案「请尽快完成支付」
2. 收货方式（条件）
3. 收货地址 / 自提地址（条件）
4. 套餐卡片
5. 订单备注 / 优惠券 / 权益卡
6. 费用明细
7. 支付方式
8. **订单信息卡** `OrderDetailInfoView`：订单编号、下单时间、支付方式、订单备注
9. 底部立即支付栏（不变）

## 数据

- 结算数据：沿用 `getOrderSettlement(orderId)`
- 订单信息：`getAppOrderDetail(orderId)`（仅 `order_pay` 模式并行拉取）
- 支付方式行：未支付时展示当前选中的微信/支付宝（随 `payMethod` 更新）

## 导航行为

- `order_pay`：普通 `pop` 返回订单列表；支付成功 `replaceWithOrders`（与 `default` 相同）
- **不得**套用 `cartCheckout` 跨 Tab 逻辑
